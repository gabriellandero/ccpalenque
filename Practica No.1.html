Diseño de Algoritmo y Diagrama de Flujo.

Título: Algoritmos y problemas
Duración: 100 min.
Propósito: Identificar y analizar el esquema básico (E / P / S), en la solución de problemas.
Contextualización: Existe un algoritmo poco conocido llamado “algoritmo babilónico”, utilizado para resolver un problema matemático muy común en las escuelas primarias, vamos a analizarlo para identificar que es lo que hace. << ver >>   <<Descargar>>

Requerimientos:
1.- Software DFD
2.- El algoritmo babilónico en copias  en presentación electrónica.
5.- Calculadora
6.- Papel y Lápiz

Desarrollo:
1. El alumno analiza  el siguente algoritmo babilónico como a partir de sumas y divisiones sencillas se hace un cálculo misterioso que se va a obtener.

2. El alumno se cuestiona  de manera intuitiva traten de describir cómo funciona,  identifiquen que elementos entienden . Puede plantear las siguientes preguntas en orden de
mayor dificultad:
a) ¿Cuántas figuras distintas observas?. Indica cuales son.
b) ¿De manera intuitiva, que piensas que signifique Lea x?
c) ¿De manera intuitiva, que piensas que signifique b = x?
d) ¿Por qué piensas que la figura del rombo tiene dos flechas, una con No y otra con Si.
e) ¿De manera intuitiva que piensas que significa la expresión: ½( x / b + b)?
f) ¿De manera intuitiva, que piensas que significa: Escriba √x = b ?

3. Una vez comentadas estas preguntas , clarifica  como funciona el algoritmo resolviendo un ejercicio breve, utilizando como valor de entrada el numero 4.

4. Realizar el  proceso con la calculadora para ir aproximándose al resultado utilizando solamente sumas y divisiones.

5. El alumnos acerca observa el proceso que está realizando , ¿Qué es lo que observan que sucede en cada paso del algoritmo?
¿Qué le paso al valor de la variable b?

6. Con una idea más clara del algoritmo, los alumnos resolverán diversos ejercicios manualmente con apoyo de la calculadora siguiendo la metodología analizada en el algoritmo.

7. Demostrará de forma práctica la ejecución del algoritmo en el software DFD puntualizando las etapas de Entrada – Proceso – Salida.
